HTML

  • HTML (HyperText Markup Language) is one of the cornerstone programming languages of the modern internet, used to give a web application its structure
  • This is accomplished through the use of HTML elements
  • Elements nested within other elements are called child elements, the element they are nested within being the parent element

  • The <head> element of an HTML page contains information (metadata) which helps the browser render the page correctly
  • Metadata comes in the form of child elements such as <title>, which sets the name of the website in search engines and in the title bar, or <style>, which defines the CSS styles used by the website
  • Elements within the <head> are not visible when a user visits the page
  • The <body> element of an HTML page contains content which will be shown to a visiting user
  • Also contains child elements, such as <h1>, which creates headings, <ul>, which creates bulleted lists, or <li>, which creates items within lists

CSS

  • CSS (Cascading Style Sheets) is a programming language used to give a web application style, such that it can be more organized and aesthetically pleasing
  • A CSS rule is composed of a selector followed one or more declarations
  • The selector defines what elements or attributes to which the rule will apply
  • A declaration is made up of two components; the CSS property, and the the value of that property to be applied
  • For example, <p> {color: red;} will turn all text within <p> elements red
  • In this case, "color" is the property, and "red" is the value

  • There are three ways to style a webpage with CSS
  • In-line CSS styles a single element directly in HTML
  • This style attribute will only apply to the specified element
  • For example, <span style = "color: #ff0045;">example<span> will turn the text within ruby red
  • Internal CSS styles the entire page via CSS rules written in the <head> element
  • Applies style attribues to all HTML elements which fall under the categories you assign
  • External CSS styles the entire page, as per Internal CSS, but with the rules contained in an external CSS file
  • The most common form of CSS, as it allows developers to keep all their CSS rules in a separate file, making design changes easier
  • Connected to the HTML file via a <link> element in the <head>

Git

  • GitHub is the web interface for Git, a version control software which allows multiple people to make separate changes to a program at the same time without overwriting each others' work
  • A repository (repo) is a folder on GitHub which contains all the files for your project and their revision history
  • A branch is a workspace within a repository where you can make changes to code or other files that won't immediately overwrite the live version
  • A Markdown (.md) file is the type of file in which changes are saved and formatted in GitHub
  • Markdown allows for writing and styling text and easy conversion to HTML
  • Committing is the process of saving a new version of the file you're working on
  • Adding, editing, or deleting a file is a commit and is added as a change to your current branch
  • A Pull Request (PR) is when a user working on a repository branch asks for changes from one branch to be merged into another branch
  • Pulls requests are often used to discuss and review changes with others involved in the project
  • When a pull request is approved, the changes are pulled in and merged into a different branch, generally moving on to be deployed to the live version of the project
  • An issue is a way to report on new tasks or content fixes
  • Progress is tracked on project board and the issue can be closed once completed / dealt with

The following are some useful Git commands; note that all Git commands must be preceded by "git"


  • git status -> shows information regarding the current active repository & branch
  • git checkout -b TITLE
  • checkout -> used to move from the current working branch to a different branch
  • -b -> used to create a new branch
  • TITLE -> new branch will have the name TITLE
  • git add -A
  • add -> stages a new file, i.e. it will be included when committing changes
  • -A -> stages all files in their current states
  • git commit -m "made a test branch titled TITLE" -> commits all currently staged files, tagged with the message "made a test branch titled TITLE"
  • git pull origin main
  • pull -> copies a branch's modifications to the local environment
  • origin -> indicates source from which you will pull
  • main -> the source branch we are currently pulling from
  • git push origin TITLE
  • push -> sends staged files to branch indicated by origin
  • TITLE -> branch indicated by origin, to which committed changes will be sent

JavaScript

  • JavaScript is a programming language that adds interactivity to a website, often in the form of buttons, data entry, dynamic styling, animations, and more
  • There are a variety of user-created tools built on top of JavaScript, such as Application Programming Interfaces (APIs) built into web browsers, or third-party APIs that allow developers to incorporate functionality from other websites
  • Similar to external CSS, JavaScript files can be linked to an HTML document using the <script> element

The following are some core components of JavaScript and details on how they work / are used


  • Variables are containers which store values, declared with the let or var keyword, followed by a name, and it can then be assigned a value
  • An example of a variable is a string, which can hold a sequence of text
  • JavaScript is a dynamic language, meaning variables are not directly associated with a specific value type, and can be assigned or reassigned values of all variable types
  • JavaScript is also a weakly typed language, meaning it will convert mismatched data types where applicable
  • Operators are symbols which perform a mathematical operation on two values or variables to produce a resulting value, such as addition (+), subtraction (-), or multiplication(*)
  • Control flow is the order in which a computer executes code
  • In the case of JavaScript, code is executed from top to bottom, but can be interrupted or otherwise altered through conditional statements
  • Conditionals test whether or not an expression is true, often in the form of an if-else statement
  • Functions are packages of code / functionality that can be reused
  • Some functions are built into the programming language, such as querySelector() or alert(), while others are written by developers
  • Events are another way to describe activity in the browser, and interactivity requires event handlers, which identify browser activity and run code in response
